Hippo Channel Manager Page Management Support

This project provides some add-on features like the following:

  • DocumentCopyingPageCopyEventListener handling PageCopyEvent propagated by Hippo Channel Manager, in order to copy all the linked documents by the page and its components together, when a page is being copied from a channel.
  • DocumentManagementService component to provide document/folder workflow operations.

DocumentCopyingPageCopyEventListener component

org.onehippo.forge.channelmanager.pagesupport.channel.event.DocumentCopyingPageCopyEventListener component copies linked documents in a page and its components from the source channel to the target channel on the event of page copying in the channel manager.

DocumentCopyingPageCopyEventListener can be extended for some custom needs.
In version 1.2.0, the hooks onBeforePageCopyEvent and onAfterPageCopyEvent have been introduced. See at the custom configuration page.

DocumentManagementService component

org.onehippo.forge.channelmanager.pagesupport.document.management.DocumentManagementService component is provided to manage document/folder workflows more easily.

Demo Application

Demo Application with Hippo CMS 10

You can build and run a demo application, which contains all the examples explained in this site documentation. The demo application is present in the demo submodule.

Testing with Demo Application

The demo application consists of many different channels. This basic testing steps will show how a page in the source (English / US) channel can be copied to the target (Dutch / Belgium) channel with the linked document content together as an example.

Browse the content folder in the source (English / US) channel. You will see three published documents there.

Now, browse the target (Dutch / Belgium) channel. You won't see any documents there yet.

Let's select the Channels perspective. You will see many channels. We are going to copy a page from the source (English / US) channel to the target (Dutch / Belgium) channel. So, please make sure that the target (Dutch / Belgium) channel should have a preview mode configuration.

Note: if a channel doesn't have a preview mode configuration, it won't be available in the dropdown of the Page Settings dialog as a target channel when copying a page.

Now let's edit the source (English / US) channel. And click on the Pages link and create a new page called "test".

Add three Simple Content catalog components in the "test" page. And link to each of the documents under the "content" folder and save the page.

Click on the Page Settings link and select the target (Dutch / Belgium) channel to copy the "test" page.

Note: if you don't see a target channel in the dropdown, please make sure that the target channel should have a preview mode configuration in prior.

Open the target (Dutch / Belgium) channel and select the copied page, "test".

You will see the copied page, "test", with the copied documents together.

Also browse the content folder of the target (Dutch / Belgium) channel. You will see all the three documents copied from the source (English / US) channel.

Note: All the linked documents from the copied page are copied together into the target channel content folder. However, the copied documents are taken offline after copying. Editors are responsible for publishing those copied documents afterward.
